# Многосессионный LLM чат‑сервис

Чат‑сервис на базе LLM с поддержкой нескольких сессий, включает бэкенд на FastAPI, хранилище PostgreSQL, сервисный слой LLM и Streamlit UI.

## Архитектура

```text
Браузер
  -> Streamlit UI
  -> FastAPI REST API
  -> Слой ML-провайдера: MockLLM или GroqLLM
  -> PostgreSQL: пользователи, API-ключи, чат-сессии, история чатов
```

## Основные возможности

- Несколько чат-сессий на одного пользователя
- История чатов хранится для каждой сессии
- Обычные и потоковые (streaming) эндпоинты чата
- Опциональная работа с реальным LLM через API, совместимый с Groq
- Инициализация модели через lifespan
- Строгие схемы запросов и ответов на Pydantic
- Streamlit UI с индикаторами загрузки и понятными сообщениями об ошибках бэкенда
- Запуск через Docker Compose одной командой

## Быстрый старт

Создайте локальный файл окружения:

```bash
cp .env.example .env
```

Запустите весь проект:

```bash
docker compose up --build -d
```

Откройте:

- Документация API: [http://localhost:8000/docs](http://localhost:8000/docs)
- Проверка состояния: [http://localhost:8000/health](http://localhost:8000/health)
- Streamlit UI: [http://localhost:8501](http://localhost:8501/)

Остановить проект:

```bash
docker compose down
```

## Использование UI

1. Откройте [http://localhost:8501](http://localhost:8501/).
2. В боковой панели введите имя пользователя и email.
3. Нажмите `Continue`.
    - Если пользователь уже существует, UI загрузит его по email.
    - Если пользователя нет, UI создаст нового.
    - Также UI создаст внутренний API-ключ для вызовов бэкенда.
4. Нажмите `New chat`.
5. Отправьте сообщение в поле ввода чата.
6. Нажимайте на предыдущие чаты в боковой панели, чтобы загрузить их историю.

В разделе `Developer info` отображаются внутренние значения, такие как user id, API-ключ, активная сессия и статистика последнего ответа.

Для контроля ресурсов UI отправляет в модель только последние 12 видимых сообщений как контекст. Используется `max_tokens=1000` для запросов чата.

Статистика последнего ответа показывает:

- Количество символов контекста, отправленных на бэкенд
- Количество символов в ответе модели
- Количество сообщений, отправленных как контекст

## Конфигурация

Все параметры выполнения считываются из `.env`. Файл `.env.example` — это шаблон, его можно скопировать в `.env`.
В проекте по умолчанию используется Groq как провайдер, он предоставляет бесплатный API. Вы можете создать его самостоятельно и затем скопировать API в файл .env.
